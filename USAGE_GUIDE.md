## システム概要

本システムは、輸出契約書を自動分析し、日本の外為法と米国EARの適用判断を支援するAIシステムです。

### 主な機能

- ✅ **契約書自動分析**: PDFから品目、仕向地、需要者情報を抽出
- ✅ **外為法判断**: リスト規制、キャッチオール規制の自動判定
- ✅ **米国EAR判断**: ECCN番号、カントリーチャート、制裁リストの確認
- ✅ **リスク評価**: 総合的なリスクレベル（高/中/低）の判定
- ✅ **推奨アクション**: 具体的な対応手順の提示
- ✅ **チャット相談**: 輸出管理に関する質問に回答
- ✅ **データ管理**: CSV形式で規制リストを管理

## 起動方法

### 1. 依存関係のインストール

```bash
pip install -r requirements.txt
```

### 2. 環境変数の設定

`.env`ファイルが既に作成されています。API keyが正しく設定されているか確認してください。

### 3. アプリケーションの起動

```bash
streamlit run app.py
```

ブラウザで `http://localhost:8501` が自動的に開きます。

## 機能説明

### 1. 契約書分析タブ

#### PDFアップロード
1. 「契約書アップロード」セクションでPDFファイルを選択
2. 「分析開始」ボタンをクリック
3. AIが自動的に以下を分析：
   - 品目名の抽出
   - 仕向地の特定
   - 需要者情報の確認
   - 外為法の適用判断
   - 米国EARの適用判断
   - リスクレベルの評価

#### 手動入力
PDFがない場合は、「または、契約情報を手動で入力」を展開して情報を入力できます：
- 品目名
- 仕向地（輸出先国）
- 需要者（エンドユーザー）
- 用途
- 契約金額
- 納期

#### 分析結果の確認

**リスクレベル表示**
- 🔴 **高**: 詳細な審査と許可申請が必要な可能性が高い
- 🟡 **中**: 追加確認と慎重な判断が必要
- 🟢 **低**: 重大な懸念は検出されていない

**抽出された契約情報**
- PDFから自動抽出された情報を表形式で表示

**AI分析詳細**
- 外為法判断フロー（A〜E）
- 米国EAR判断フロー（A〜E）
- 総合判定
- 必要な手続き

**推奨アクション**
- 具体的な対応手順のチェックリスト

**ダウンロード**
- 分析結果のテキストファイル
- 詳細レポート（契約情報+分析結果+推奨アクション）

### 2. チャット相談タブ

輸出管理に関する質問をAIに聞くことができます。

#### 使用例

**質問1**: 「グループA国とは何ですか？」
**質問2**: 「少額特例の適用条件を教えてください」
**質問3**: 「ECCN番号5A002はどのような品目ですか？」
**質問4**: 「中国向け輸出で注意すべき点は？」

#### 会話履歴
- 過去の質問と回答が自動的に保存されます
- 各質問をクリックして内容を確認できます

### 3. データ管理タブ

規制に関するデータを管理・確認できます。

#### サンプルデータ

**ECCN番号リスト**
- 主要なECCN番号と品目情報
- 品目名で検索可能

**カントリーグループ**
- グループA国（旧ホワイト国）のリスト
- 懸念国・国連武器禁輸国の情報
- 国別統計（グループA国数、懸念国数）

**エンティティリスト（サンプル）**
- 制裁対象企業のサンプルデータ
- 実際の取引では最新の公式リストを確認してください

#### カスタムデータのアップロード
- 独自のCSVファイルをアップロードして管理できます
- ECCN番号リスト、カントリーリスト等を更新可能

## 使用例

### 例1: 半導体製造装置の中国向け輸出

#### シナリオ
- 品目：半導体製造装置（エッチング装置）
- 仕向地：中国
- 需要者：Example Semiconductor Corp
- 用途：民生用半導体の製造
- 契約金額：5,000万円

#### 分析プロセス
1. 契約書PDFをアップロード（または手動入力）
2. 「分析開始」をクリック
3. システムが自動的に分析：
   - 品目がリスト規制該当の可能性を検出
   - 仕向地が中国であることを確認（非グループA国）
   - 需要者のエンティティリスト照合
   - 用途の確認（軍事転用の懸念）
   - 米国EAR適用の可能性を検証

#### 結果表示
- **リスクレベル**: 高
- **外為法判断**: リスト規制該当の可能性、キャッチオール規制要確認
- **米国EAR判断**: EAR対象品目の可能性、エンティティリスト要確認
- **推奨アクション**:
  - ✅ 正確な該非判定を実施
  - ✅ エンドユーザーの詳細情報を確認
  - ✅ 制裁者リストとの照合を徹底
  - ✅ 経済産業省へ事前相談

### 例2: 研究用試薬のドイツ向け輸出

#### シナリオ
- 品目：研究用試薬（一般化学品）
- 仕向地：ドイツ
- 需要者：ミュンヘン大学
- 用途：基礎研究
- 契約金額：50万円

#### 分析プロセス
1. 契約情報を手動入力
2. 「分析開始」をクリック
3. システムが自動的に分析：
   - 品目がリスト規制非該当の可能性
   - 仕向地がグループA国であることを確認
   - 用途が基礎研究であることを確認
   - 少額特例適用の可能性を検証

#### 結果表示
- **リスクレベル**: 低
- **外為法判断**: リスト規制非該当の可能性、グループA国向け
- **米国EAR判断**: 米国原産品でなければEAR規制対象外の可能性
- **推奨アクション**:
  - ✅ 該非判定を実施
  - ✅ 米国原産品の有無を確認
  - ✅ 基礎科学分野研究の許可例外適用を検討

## トラブルシューティング

### 問題1: PDFからテキストが抽出できない

**原因**
- PDFが画像形式（スキャン文書）
- PDFが暗号化されている
- PDFのフォーマットが複雑

**解決方法**
- OCR処理済みのPDFを使用
- 手動入力機能を利用
- テキスト形式で情報をコピー＆ペースト

### 問題2: 分析結果が不正確

**原因**
- 契約書に必要情報が不足
- 専門用語や略語が多い
- 情報が複雑で抽出が困難

**解決方法**
- 手動入力で正確な情報を提供
- チャット機能で詳細を確認
- 専門家に相談（CISTEC、経済産業省）

### 問題3: エラーメッセージが表示される

**原因**
- OpenAI API keyが無効
- ネットワーク接続の問題
- ファイル形式が非対応

**解決方法**
- `.env`ファイルのAPI keyを確認
- インターネット接続を確認
- 対応形式（PDF、TXT）のファイルを使用

### 問題4: サンプルデータが表示されない

**原因**
- `sample_data`フォルダが存在しない
- CSVファイルが破損

**解決方法**
- `sample_data`フォルダが正しく配置されているか確認
- CSVファイルの文字コードを確認（UTF-8推奨）

## 注意事項

### ⚠️ 重要な免責事項

1. **本システムは参考情報を提供するものであり、法的助言ではありません**
2. **最終的な判断は必ず専門家や関係当局に確認してください**
3. **AIの分析結果には誤りが含まれる可能性があります**
4. **実際の輸出管理業務では、最新の法令・通達を確認してください**

### 推奨事項

- ✅ 該非判定は専門家（CISTEC等）に依頼することを推奨
- ✅ 不明な点は経済産業省に問い合わせ
- ✅ 米国EARについては米国BISまたは専門弁護士に相談
- ✅ 定期的に規制リストを更新
- ✅ 輸出管理内部規程（CP）を整備

### セキュリティ

- 🔒 APIキーは絶対に第三者と共有しない
- 🔒 機密情報を含む契約書は慎重に取り扱う
- 🔒 分析結果は適切に保管・管理する
